# contract: v1.0.1

Record outcome AFTER an infrastructure operation completes.

Required inputs:
- prescription_id (from prescribe)
- exit_code (0 success, non-zero failure)


Optional inputs:
- actor (type, id, origin; should match prescribe actor)
- actor.version, actor.skill_version
- artifact_digest — SHA-256 of the artifact actually applied (format: "sha256:<hex>").
  Strongly recommended whenever the prescription had a raw_artifact.
  Omitting this disables artifact drift detection and fires a protocol_violation/report_without_digest signal.
  Compute with: sha256sum <file> or echo -n "<content>" | sha256sum
- session_id, operation_id, span_id, parent_span_id


Terminal outcome rule:
- Every prescribe must end with exactly one report, including failed/aborted attempts.
- Retries require a new prescribe/report pair for each attempt.


Rules:
- Always report failures; do not hide non-zero exit codes.
- Do not report twice for the same prescription_id.
- Do not report another actor's prescription_id.
- If prescription_id is lost, call prescribe again before execution.
- Actor identity should match the original prescribe actor.
- Include actor.skill_version for behavior slicing.


Returns:
- ok
- report_id
- signals (optional)
- error (if ok=false)
