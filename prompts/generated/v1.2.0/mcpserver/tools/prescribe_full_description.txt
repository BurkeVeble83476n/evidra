# contract: v1.2.0

Record full artifact intent BEFORE an infrastructure operation that creates, modifies, or deletes resources. Use this tool when you have the actual artifact bytes and want native detector coverage plus artifact drift tracking.

Required inputs:
- tool
- operation
- raw_artifact
- actor (type, id, origin)


Recommended actor fields for behavior slicing:
- actor.version
- actor.skill_version


Optional correlation inputs:
- session_id, operation_id, attempt
- trace_id, span_id, parent_span_id
- scope_dimensions


Pre-call checklist:
- tool/operation must be non-empty.
- raw_artifact must contain the artifact or command content being executed.
- actor.type/actor.id/actor.origin must be present.
- actor.skill_version should be set to this contract version.
- Keep session/operation/trace identifiers stable within one task.


When to call:
- Must call `prescribe_full` for mutate operations when artifact bytes are available.
- Skip for read-only operations (get/describe/logs/plan/show/status/template).

Returns:
- prescription_id (required for report)
- risk_inputs, effective_risk
- artifact_digest, intent_digest
- resource_shape_hash, operation_class, scope_class


Interpretation:
- `effective_risk` is the summarized decision hint.
- `risk_inputs` contains the per-source risk observations behind that summary.

CRITICAL: If prescribe_full returns ok=false, do not execute the infrastructure command. Investigate the error before retrying.
